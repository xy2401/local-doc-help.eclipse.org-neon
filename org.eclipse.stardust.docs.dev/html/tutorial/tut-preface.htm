Tutorials

The tutorials provide practical introductions into the concepts of Stardust functionality.

Audience

Analysts, developers as well as end users find instructions and use cases appropriate to their work in these tutorials.

Contents

The tutorials guide you through the following use cases:

  • The Support Case Example - Eclipse Modeler - a simple case study on how to model and then automatize a simple "support case" process in a fictitious company called ACME. This introduces the Stardust Process Eclipse modeler and the Stardust runtime environment.
  • The Support Case Example - Web Modeler - a simple case study on how to model and then automatize a simple "support case" process in a fictitious company called ACME. This introduces the Stardust Process Modeling perspective and the Stardust runtime environment.
  • Mashing up Angular based UI into Stardust Portal - this example describes the process how to mashup an angular based UI into the Stardust Portal.
  • Scoped Participant Tutorials:
    • The Scoped Participant Example - is a simple use case created to explain how to develop a multi-scoped organizational structure and work with multi-scoped participants.
    • Use Case - Delegating to a User in Another Target Department - a simple use case example on how to delegate a user to a user in another target department.
  • Defining and Running Reports Tutorials:
    • Creating and Running a Simple Report Definition - a tutorial guiding you through the process of creating a simple report
    • Creating and Running a Report with Table Layout and a computed Column - an example on how to create a report definition using a table layout, computed columns, filtering as well as the scheduling functionality
    • Creating a Composite Report - an example on how to create composite reports
  • Using a Configuration Variable to Control a Process Flow - This tutorial guides you through a use case, where a configuration variable is set to control if an activity is executed or skipped during runtime.
  • Performing Quality Assurance on interactive Activities - A tutorial to demonstrate how to perform quality assurance on interactive activities.
  • Managing Unstructured Workflows
    • Spawning Sub-processes - tutorial to describe how spawning of a root process instance creates a sub-process and copies data associated with it.
    • Extracting Pages Tutorial - tutorial to demonstrate how to extract pages from a document attached to a process instance and create a new process instance containing the extracted pages
    • Aborting and Starting Processes - tutorial to help you understand how to abort existing process instance and start another process instance, to check linking between them, and to check whether data is copied from source process instance to target process instance.
    • Aborting and Joining Processes - tutorial to help you understand how to abort existing process instance and join to another active process instance and to check linking between them.
  • Document Triage Tutorial - This tutorial guides you through the process of triaging incoming documents.
  • Creating Correspondence - This tutorial guides you through the process of creating correspondence using predefined document request data.
  • Working with multiple Models in one Audit Trail - This tutorial demonstrates how to work with multiple models in one audit trail database.
  • Creating a Worklist Filter - This tutorial helps you to create and apply worklist filter for portal configuration.
  • Using JMS Application and JMS Trigger - This tutorial helps you to work with JMS application and JMS trigger.
  • Defining a Process Interface with dynamic Process Selection - This example describes a fictional product purchase process using dynamic process selection.
