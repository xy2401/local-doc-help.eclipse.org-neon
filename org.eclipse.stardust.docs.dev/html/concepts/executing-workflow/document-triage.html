Document Triage

Efficiently sorting, tagging, and distributing incoming documents is a requirement of all implementations that utilize Stardust Document Management. The Document Triage functionality enables you to quickly have look at a document and take a decision depending on the requirement. The Portal Document Triage offers:

  • an easy way to split documents
  • search for existing in-flight Processes to associate documents (see Document Rendezvous)
  • an Document Triage screen to simplify and streamline triage for incoming documents, where users can select a Business Object from a list to populate the Process Data panel
  • start new Process Instances with incoming Documents

Common Use Cases

Common use cases are the following:

  • A user takes the incoming documents and applies a new process with them. For example he starts a new loan process with selected documents.
  • A loan application has been performed, but the customer forgot to send the statement with it. A correspondence can be started to send to the customer as notification that the statement has been forgotten to send. The activity instance goes in hibernated state until the required document comes in.
  • Once a new document enters a Document Triage, the user can search for all processes using a specific Business Object. The resulting process instances are listed and the user can drag the document to the process instance(s) in question.

Triaging Documents in the Stardust Portal

The Document Triage screen in the Stardust Portal provides an ergonomic and efficient UI to:

  • View incoming Documents during data entry
  • Select a Business Object from the list to populate the Process Data panel
  • Split incoming documents
  • Search for existing in-flight Processes to associate documents (Document Rendezvous)
  • Start new Process Instances with incoming Documents

Please refer to chapter Triaging Documents in the End User Handbook for details.

Example Use Cases

The following steps describe a typical use case workflow:

  1. Documents enter Stardust via Scan or E-mail
  2. The Index operator opens an Index Activity (1st Manual Activity)
  3. View Document(s) and find key pieces of Data (like Member #, Last Name, etc.)
  4. Select Participant Record
  5. Enter Key Process Data
  6. Look-up Against Record Keeping System
  7. Select Participant Record
  8. Search Stardust to determine if any In-flight Process is waiting for any of these documents:
    • Yes: add Document(s) to in-flight process
    • No: Continue

  11. If necessary, split Documents
  12. Set a Document Type for Process Attachments
  13. Set Specific Documents
  14. Complete Activity when all documents have been triaged
