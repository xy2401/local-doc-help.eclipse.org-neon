Creating the Departments and Users in the Stardust Portal

This chapter guides you through the following steps to prepare the delegation scenario:

  • Starting the Portal
  • Creating the Users
  • Creating the Departments
  • Associating the Users

Starting the Portal

To be able to perform the delegation scenario in the Stardust Portal, start your server and deploy the model. For information on how to deploy a model, please refer to the chapter Deploying a Workflow Model of the Deployment guide.

Start the Stardust Portal as described in the chapter Logging in the Stardust Portal of the End User Handbook. Login as administrator (motu/motu). You can use the Shift-F8 short key.

Creating the Users

First create the users, who will be assigned to the performing roles or organizations in different departments:

  1. Switch to the Administration Perspective.
  2. In the All Users part of the Participant Manager view create the following users:
    • Hans Nase
    • Frank Cityman
    • Daisy Duck
    • Robert Doodle
    Choose passwords and accounts as you like.


Figure: Create new Users.

Creating the Departments

Now create two departments, where the activity will be delegated from and to:

  1. In the Participant Tree, right-click NationDivision and choose Create Department.
  2. Enter a department with name and ID Germany.

  3. In the same way create a department USA.

Associating the Users

Now associate the users with roles and organizations in the different departments:

  • Associate user Nase, Hans with department Germany and the team leader NationalDirector in this department.
  • Associate user Doodle, Robert with the role Engineer in department Germany.
  • Associate user Cityman, Frank with department USA and the team leader NationalDirector in this department.
  • Associate user Duck, Daisy with the role Engineer in department USA.
