Extracting Pages

The functionality to extract pages from a document added to a process instance and pass those pages to a new process that will be started.

  • Use Case
  • Version Behavior of New Documents

For details on extracting pages, refer to chapter Extract Pages in the End User Handbook.

Common Use Case

Consider a scenario of an organization that often receives work via fax or regular mail containing multiple requests for a single client. These documents enter Stardust as single multi-page TIFF documents and are attached to a single process instance.

The extracting pages functionality allows operators in the Stardust Process Platform to extract pages from these TIFF files and start a process instance for each separate request. Additionally, these process instances provide a linkage so that business units can track the separate requests as a single entity.

The workflow would be as follows:

  • The company receives documents via fax.
  • Via a Scan client, the documents are scanned into a multi-page TIFF file into a process definition.
  • The operator logs into the Stardust Portal and locates the process definition created via the Scan operation.
  • He activates an activity in the created process instance and opens the multi-page document in the TIFF document viewer.
  • Now he clicks the Extract Pages icon in the Bookmarks View toolbar to launch the Extract Pages dialog.
  • The Start Process column in the dialog provides a drop down list with all processes he has the ability to manually start.
  • The Document Type drop down list contains all document types.
  • In case the selected Document Type is linked to multiple document types in the selected Start Process column, the Document Data list is enabled.
  • The operator selects to delete the extracted pages, copy annotations and bookmarks and to copy process data.
  • Clicking OK starts the selected process.
  • Now a new process is started as sub-process of the original process.
  • This process is displayed in the Process History of the original process.
  • The extracted pages should be deleted in the TIFF document attached to the original process.
  • The added document fragments in the newly started process should contain the extracted pages defined in the dialog. Annotations, bookmarks and data should be copied.

Version Behavior of New Documents

Extracting pages creates a new document version for the target document. A version comment is automatically applied to the target document as follows:

  • Target Document Version Comment: 
    Document created by extracting pages from source_doc_id.

In case Delete Pages is marked in the Extract Pages dialog for these pages, a new version of the source document is created and a version comment is automatically applied to the source document as follows:

  • Source Document Version Comment: 
    Extracted pages x-y to document target_doc_id and started process process_id.
