Setting the Scene

This chapter introduces you to Stardust from a pragmatic point of view. This tutorial explains how to model a simple business process and deploy it for improved automation and management. The example focuses on a process modeled for a customer call center in a fictitious company called ACME.

Goals

In this example, ACME is an independent software vendor and needs to set up a support management process. In the customer call center, agents are put all incoming support incidents on record. The engineering staff is responsible for analysis of the recorded calls and for providing a solution to each of them.

Prerequisites

Using this tutorial for hands-on learning, requires a prior installation of Stardust on your system. Installation issues are covered in the Stardust Installation Guide.

Familiarity with basic concepts of workflow modeling is recommended, though not required, for understanding this chapter. A more theoretical introduction to workflow modeling with Stardust is provided in the Modeling Guide.

Plan

Once you execute this tutorial, you can:

  • Create a model
  • Create a process definition
  • Define activities
  • Define transitions between activities
  • Define applications
  • Determine process data
  • Assign roles, which associate participants to workflow activities
  • Create events
  • Deploy your model to the Stardust Audit Trail and make it accessible for execution clients (runtime environment)
  • Manage users in the Stardust Administration Perspective
  • Accomplish a work item in a user's worklist
