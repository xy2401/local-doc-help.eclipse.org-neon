3.18.2. Executing and analyzing manual tests

Once you have created a manual Test Case (Section 3.18.1, “Creating manual tests”), you can execute it.

  1. Start the AUT (Section 3.20.2, “Starting the AUT”).

  2. Start the Test Suite containing the manual Test Steps (Section 3.20.3, “Starting, stopping and pausing Test Suites and Test Jobs”).

  3. For each manual Test Step in the test, a dialog will appear with the descriptions and data you entered for the action to perform and the expected behavior.

  4. Execute the Test Step as described, enter a comment if necessary, then click either ”passed” or ”failed” in the dialog to move to the next Test Step.

    If the Test Step fails, then an automatic screenshot will be taken (Section 3.28.9, “Test result preferences”), and any Event Handlers (Section 3.26, “Dealing with errors in tests: Event Handlers”) for this Test Case will be activated.

  5. Once the dialog has been closed, the next Test Step dialog will appear.

  6. At the end of the test, you can see the test result report like any other automated test (Figure 3.29, “Test Result View”).
