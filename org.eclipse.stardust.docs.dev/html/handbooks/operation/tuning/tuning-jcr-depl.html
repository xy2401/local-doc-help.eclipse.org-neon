Optimizing the Performance of secure JCR Deployments

Setting up secure JCR sessions produces a high number of data base accesses to initialize authorizations. Using the 2nd-level caching feature provides most of the required information and could reduce the load from the data base. Thus we recommend to enable 2nd-level caching for production deployments.

Refer to chapter Retrieving Entities from In-Memory Cache for details on how to enable and configure 2nd-level caching.
