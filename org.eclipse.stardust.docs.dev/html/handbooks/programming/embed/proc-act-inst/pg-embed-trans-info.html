Retrieving Information on Transition Steps and Targets

Class TransitionInfo serves as base class to provide functionality to retrieve information on transition steps and targets.

  • Retrieving the Source Activity Instance OID
  • Retrieving the OID of the Model containing the Activity that represents the Target
  • Retrieving the Runtime OID of the Activity that represents the Target
  • Retrieving the Id of the Activity that represents the Target
  • Retrieving the Name of the Activity that represents the Target
  • Retrieving the Id of the process definition containing the Activity that represents the Target
  • Retrieving the Name of the process definition containing the Activity that represents the Target
  • Retrieving the Id of the Model containing the Activity that represents the Target
  • Retrieving the Name of the Model containing the Activity that represents the Target

Retrieving the Source Activity Instance OID

The value, which the method getActivityInstanceOid() retrieves, is required to perform the actual transition. It is different for transition target and transition step in the following way:

  • TransitionTarget: the activity instance from where the ad-hoc transition is performed
  • TransitionStep: the OID of the activity instance that started the subprocess
  • TransitionStep and there is no corresponding activity instance (no subprocess has started yet): -1
   public long getActivityInstanceOid()
   {
      return activityInstanceOid;
   }

Retrieving the OID of the Model containing the Activity that represents the Target

Method getModelOid() retrieves the OID of the model containing the activity that represents the target of the current operation. This value of the OID is necessary to perform the actual transition.

   public long getModelOid()
   {
      return modelOid;
   }

Retrieving the Runtime OID of the Activity that represents the Target

Method getActivityRuntimeOid() retrieves the runtime OID of the activity that represents the target of the current operation. This value is necessary to perform the actual transition. Together with the model OID is used to uniquely identify a specific activity in the model repository.

   public long getActivityRuntimeOid()
   {
      return activityRuntimeOid;
   }

Retrieving the Id of the Activity that represents the Target

The following method is used to retrieve the Id of the activity that represents the target of the current operation. It returns a String containing the Id of the activity.

   public String getActivityId()
   {
      return activityId;
   }

Retrieving the Name of the Activity that represents the Target

Method getActivityName() retrieves the name of the activity that represents the target of the current operation. It returns a String containing the name of the activity.

   public String getActivityName()
   {
      return activityName;
   }

Retrieving the Id of the process definition containing the Activity that represents the Target

The following method is used to retrieve the Id of the process definition containing the activity that represents the target of the current operation. It returns a String containing the Id of the process definition.

   public String getProcessId()
   {
      return processId;
   }

Retrieving the Name of the process definition containing the Activity that represents the Target

Method getProcessName() retrieves the name of the process definition containing the activity that represents the target of the current operation. It returns a String containing the name of the process definition.

   public String getProcessName()
   {
      return processName;
   }

Retrieving the Id of the Model containing the Activity that represents the Target

To retrieve the Id of the model containing the activity that represents the target of the current operation, the following method is used:

   public String getModelId()
   {
      return modelId;
   }

It returns a string containing the Id of the model.

Retrieving the Name of the Model containing the Activity that represents the Target

The following method retrieves the name of the model containing the activity that represents the target of the transition operation.

   public String getModelName()
   {
      return modelName;
   }

The method returns a string containing the name of the model.
