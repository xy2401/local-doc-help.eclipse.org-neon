Internationalization of the Modeler

Elements of the Stardust Modeler are identified via entries in their according resource bundles. The resource bundle which is used depends on the preferred language set for your browser.

Resource bundles for languages are supplied for the Stardust Modeler in the according fragment jars in the eclipse/plugins folder of your Eclipse installation. The fragment jars have the same name as the according plugins jars with a .nl_ suffix before the version number. The following fragments serve for modeler internationalization:

Internationalization of Process Model Element Names

Internationalization of process model element names, set in the modeler, can be done via native language support in the modeler directly. Translation is provided for the names of the following model elements:

  • the model itself,
  • processes,
  • activities,
  • applications,
  • roles and organizations,
  • data
  • data path and data mappings,
  • diagrams,
  • triggers and
  • event handlers.

Chapter Native Language Support of the Stardust Modeling Guide describes in detail the internationalization procedure.
