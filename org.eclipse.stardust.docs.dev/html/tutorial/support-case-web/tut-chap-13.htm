Deploying a Workflow Model

To execute the workflow, the model must be deployed to the Stardust Platform runtime environment. The audit trail contains all state changes relating to any workflow instance with a time stamp and a link to the resource that initiated the state change. So it stores for instance the information who executed which activity and when.

  1. Make sure that the server is up and running.
  2. In the My Process Models panel, right-click the model name and select Deploy (Create New Version) option.
  3. In the Deploy New Model dialog, click Deploy.
  4. Click Finish once the model is deployed.
