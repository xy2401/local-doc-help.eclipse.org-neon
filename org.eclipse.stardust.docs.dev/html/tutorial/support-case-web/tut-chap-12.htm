Adding a Start Event

Events cause a process to start. To initiate a process, a start event can be defined of different types: a performer, at specific time, an incoming mail or a JMS message can serve as a start event. The swim lane and its participant defined in the previous section would be able to initiate the support case management process. So, the CallCenter Agent role is assigned to the swim lane in which Init Data activity is defined.

  1. In the toolbar, click the Create Start Event icon and drop it to the Start Process swim lane.


    Figure: Create Start Event

  2. Specify the name as Support Call.
  3. Hover the mouse over the Support Call start event and click the Connect icon.
  4. Connect the start event with the Init Data activity.


    Figure: Start Event
